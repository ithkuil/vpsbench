Benchmarking your own VPS

Log in to your VPS (800MB RAM or less only) with ssh and run this command exactly as it is written (copy and paste into terminal window):
echo "Enter VPS provider (e.g. Amazon, Linode):"; read provider;echo $provider >prov; echo "Enter RAM (MB):"; read ram; echo $ram >ram; curl -s http://bench.willsave.me/dobench.sh | bash
Then wait for it to finish. It may take anywhere from a few minutes to several minutes.

Because the disk test uses dd and because of the way file caching works, the disk speed test will only be reasonably accurate on machines with limited RAM.
Please don't run on machines with > 800MB of RAM (I will just delete these results).

Results

Larger number are better. IP address: if this is blank then the provider is using OpenVZ or Virtuozzo. CPU: 100 - (time taken to calculate the GCD of two large numbers 50000 times in bash). Network: download speed in MB/S. Disk Transfer: 100 - (time taken to use dd to copy a 1.2GB file * 1.5)

Note that when I deployed the initial version there were a lot of issues with caching or compression throwing off the numbers for the Disk Write test, so I removed all of those results (sorry) and now I am testing a combined read and write time using the 100mb.test data file cat'd several times which is less susceptible to varying compression settings or RAM size.

Obviously this is not the ultimate set of benchmarks, but it all runs in bash so it doesn't require any software to install and can be initiated with a copy and paste, which should make it easy to collect lots of data.
